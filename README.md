# Home Inventory — Backend

Бэкенд для приложения домашней инвентаризации. Учёт вещей, локаций, документов и гарантий.

## Стек

| Слой               | Технология                                       |
| ------------------ | ------------------------------------------------ |
| Framework          | NestJS 11 + TypeScript                           |
| База данных        | MongoDB + Mongoose                               |
| Кэш                | Redis + ioredis                                  |
| Очереди            | BullMQ                                           |
| Файловое хранилище | S3 (AWS SDK v3) / MinIO                          |
| Почта              | Nodemailer + @nestjs-modules/mailer + Handlebars |
| Аутентификация     | JWT + Passport                                   |

### Структура

```
src/
├── api/          # фичевые модули (controllers + services + dto)
├── config/       # фабрики конфигов для DynamicModule.forRootAsync
├── infra/        # технические модули (database, redis)
├── libs/         # внутренние сервисы-обёртки (storage, mail)
├── shared/       # типы, утилиты, константы
├── app.module.ts
└── main.ts
```

### Локальная инфра (docker-compose)

| Сервис    | Порты                  | Назначение                 |
| --------- | ---------------------- | -------------------------- |
| `mongo`   | 27017                  | Основная БД                |
| `redis`   | 6379                   | Кэш                        |
| `minio`   | 9000 (API), 9001 (UI)  | S3-совместимое хранилище   |
| `mailpit` | 1025 (SMTP), 8025 (UI) | Локальный SMTP-перехватчик |

### Запуск

```bash
cp .env.example .env
docker compose up -d
npm install
npm run start:dev
```

UI-консоли:

- MinIO → http://localhost:9001 (minioadmin / minioadmin), создать bucket `home-inventory`
- Mailpit → http://localhost:8025 (все письма из приложения ловятся сюда)

## Функциональные требования

### Аутентификация

- Регистрация по email + пароль
- Вход по email + пароль
- JWT access token + refresh token в httpOnly cookie
- Обновление access token по refresh token
- Выход (инвалидация refresh токена)

---

### Вещи (Item)

- CRUD
- Фильтрация по категории, контейнеру, статусу
- Полнотекстовый поиск по названию и описанию
- Загрузка фото через presigned S3 URL
- Генерация QR-кода при создании
- Поиск по штрихкоду товара

---

### Контейнеры

- CRUD
- Иерархическая структура (комната → шкаф → полка)

---

### Категории

- CRUD
- Иерархическая структура (Электроника → Ноутбуки)

---

### Документы

- Прикрепить файл к вещи (чек, гарантийный талон, инструкция)
- Тип документа (receipt / warranty / manual / other)
- Дата окончания гарантии

---

### Напоминания

- Cron-job раз в день — проверка истекающих гарантий
- Email уведомление за 30, 7 и 1 день до истечения
- Дедупликация повторных отправок

---

### Фоновые задачи (BullMQ)

- Генерация QR-кода после создания вещи
- Ресайз фото после загрузки
- Отправка email уведомлений
- Генерация PDF отчёта для страховки

---

### Кэш (Redis)

- Кэширование списка вещей
- Инвалидация при изменении данных
- Кэширование деревьев контейнеров и категорий

---

### Экспорт

- PDF отчёт — список вещей с фото и ценой
- Асинхронная генерация — клиент получает jobId и поллит статус
- Готовый файл сохраняется в S3
